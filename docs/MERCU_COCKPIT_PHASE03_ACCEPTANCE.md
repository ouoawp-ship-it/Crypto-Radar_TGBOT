# Mercu 风格驾驶舱 V2 · Phase 3 验收记录

## 1. 阶段结论

Phase 3 已在本地完成“信息聚合 + 结构化 AI 决策”闭环，建立在 Phase 0–2 的统一市场快照、雷达、资金中心和单币证据工作台之上。实现遵守以下边界：

- 不复制 Mercu 或其他第三方源码、品牌和受限正文；
- 资讯仅保留可公开访问的官方链接、结构化事实和本系统规则推断；
- AI/Agent 不伪造价格、资金、排名或来源，不把缺失值显示成 `0`；
- 方向性结论必须通过数据状态门禁，证据不足时返回“数据不足”；
- 不新增自动交易、下单、回测或已删除的结构突破/结构复盘功能。

## 2. 信息中心

### 数据与存储

- `NewsEventStore` 使用 SQLite 持久化标准化事件和币种关联；
- 事件字段包含稳定 ID、类型、重要度、语言、发布时间、来源、来源链接、授权状态、规则版本和关联币种；
- 标题执行长度与字符清洗，外链仅允许 HTTPS 且限制到已批准官方域名；
- 同一主题按确定性规则聚类，保留所有可追溯来源链接；
- 官方公告写入失败不阻断 Telegram 雷达主循环。

### 页面能力

- `/info` 提供来源/授权通道、关键词/币种/类型/重要度筛选、事件聚类和原文链接；
- 高重要度事件的展开内容严格分为“已知事实”和“规则推断”；
- 未授权来源明确显示不可用，不以抓取或复制正文方式补齐；
- 包含加载、错误、空状态和 390px 移动端布局。

## 3. AI 决策中心

### 规则优先 Agent

- 全局 Agent：市场广度、现货/合约主动资金与总体状态；
- BTC/ETH Agent：主流币证据与数据状态；
- 异常候选 Agent：基于已有雷达候选排序，不生成虚构候选；
- 消息 Agent：基于已入库官方事件及其重要度；
- 所有数字由程序格式化后注入，`llm_generated=false`；
- 每条结论记录生成时间、过期时间、规则版本、模型信息、证据引用、反证引用和免责声明。

### 安全门禁

只有核心市场事实为 `ready` 时才允许生成方向性表达。`degraded`、`empty` 或关键证据不足时：

1. 状态固定为 `insufficient_data`；
2. 不输出看多/看空或买卖建议；
3. 页面展示缺失证据和数据状态；
4. 仍可跳转雷达、单币、信息来源和 Telegram AI 深链进行人工复核。

## 4. API 契约

- `GET /public-api/info/feed`
- `GET /public-api/agents/overview`
- `GET /public-api/health` 增加信息/Agent 聚合健康状态

接口统一使用公开白名单投影、固定 schema 版本、长度边界和脱敏错误。资讯与 Agent 使用独立缓存，冷启动上游失败时返回可解释的降级结构，不暴露 Token、Cookie、IP 或内部异常正文。

## 5. 验收证据

- Python 单元/契约测试覆盖事件标准化、聚类、授权元数据、Agent 门禁、持久化和公开路由；
- Next.js 类型检查和生产构建通过；
- Playwright 覆盖桌面信息中心、390px 信息中心、桌面 Agent 证据展开、390px Agent 页面及降级 Agent 不生成方向结论；
- 本地真实接口测得信息接口冷/热约 `4.54s / 0.24s`、约 `25 KiB`；Agent 接口冷/热约 `5.85s / 0.005s`、约 `8 KiB`；
- 桌面 1280px 与移动 390px 均无横向溢出，浏览器控制台无新增错误。

最终全量测试数量以 Phase 4 总验收记录为准。

## 6. 已知边界

- 当前资讯源以已批准的 Binance 官方公告为主，其他来源在获得明确授权前保持不可用；
- Agent 当前是确定性规则解释器，不宣称是预测模型；
- 本地没有持续积累的生产市场历史时，Agent 显示降级或数据不足是正确行为；
- 资讯聚类采用可审计规则，不承诺覆盖所有语义相似事件。
