泡泡抓币新项目最终运行说明
更新时间: 2026-05-25

一、项目定位

这是基于旧版合并脚本经验重建的精简版。
新运行入口统一使用 main.py，旧合并脚本已经从项目中清理掉。

本版不包含 admin 查询、Web/UI、自动交易，也不再保留 Windows 一键启动脚本。
后续统一用命令行运行，方便上传 GitHub 后部署到服务器。

核心文件:
- main.py: 保留的命令行兼容入口
- paopao_radar/cli.py: 命令入口和运行流程
- paopao_radar/config.py: 配置和环境变量
- paopao_radar/data_sources.py: Binance 数据源、缓存、重试、预算、熔断
- paopao_radar/radar.py: 资金雷达、启动雷达、公告机会/风险、背离逻辑
- paopao_radar/telegram.py: Telegram 推送、安全确认、去重、限流
- paopao_radar/storage.py: JSON 状态读写、原子写入、损坏文件隔离
- paopao_radar/maintenance.py: 旧状态迁移和运行垃圾清理

二、默认安全规则

直接运行:
python main.py

默认只显示 status，不会真实推送 Telegram。

任何真实推送必须同时带:
--send --confirm-real-send

真实 live 循环还必须先通过 readiness 检查。

三、本地常用命令

查看状态:
python main.py status

查看真实推送准备度:
python main.py readiness

dry-run 观察，不真实推送:
python main.py observe --duration-minutes 360 --launch-interval 180 --launch-scan-limit 40 --records 200 --top 12

快速验证一轮:
python main.py observe --duration-minutes 0 --launch-interval 60 --launch-scan-limit 5 --records 20 --top 5

发送一条 Telegram 测试消息:
python main.py telegram-test --send --confirm-real-send

确认 Telegram 收到后开启真实 live:
python main.py live --send --confirm-real-send

查看运行心跳:
python main.py runtime-status

四、服务器一键部署

服务器上建议使用一键脚本，详细步骤见 SERVER_DEPLOY.md。

最短流程:

git clone https://github.com/ouoawp-ship-it/paopao-crypto-radar.git
cd paopao-crypto-radar
bash scripts/install_server.sh

第一次运行会创建 .env.oi 并提示填写配置。填好后再运行同一个脚本，它会自动安装依赖、跑测试、生成 systemd 服务并启动。

自动清理:
程序默认每 1 小时检查一次，只清理可再生成的垃圾：__pycache__、*.pyc、旧临时文件、坏 JSON 备份、过期日志、过长推送历史和过长启动观察历史。核心状态、.env.oi、源码不会被自动删除。

五、会推送的消息类型

1. 资金雷达汇总
周期: 默认每 30 分钟。
内容: 负费率榜、综合榜、埋伏榜、动量池、新币池、值得关注、图例、数据质量。

2. 启动雷达提醒
周期: live 默认每 3 分钟扫描一次。
内容: 币种、阶段、分数、价格/OI/成交量变化、触发原因。
阶段: watching / primed / breakout / launched / failed。
同币同阶段默认冷却 6 小时。

3. 公告机会/风险
周期: 跟随 once/loop 的主扫描。
内容: Binance 公告中的 Alpha、上新、HODLer、Launchpool、Airdrop、下架、停止交易等。
dry-run 不会把公告标记为已推送，只有真实发送成功才写入状态。

4. Telegram 测试消息
只在手动执行 telegram-test --send --confirm-real-send 时发送。

六、启动瞬间捕捉逻辑

系统不是只等币已经大涨才提示，而是分层:
- watching: 有资金/成交/OI 异动，先进入观察。
- primed: 多个条件叠加，进入提前预警。
- breakout: 接近或突破短周期结构，进入确认预警。
- launched: 分数极高，认为可能已经启动，立即提醒。

这样兼顾提前预警和启动瞬间提醒。

七、外部接口压力控制

已实现:
- 请求超时
- retry/backoff
- 短缓存
- OI/K线/费率历史请求预算
- 数据源熔断
- 扫描上限
- Telegram 全局小时限流
- 模板级日限流
- 同币同阶段冷却
