# v1.87.5 生产 SLO 与动作闭环门禁

## 原因

生产复查证明原 HTTPS 验收只限制公开情报响应大小，并未把既定的 800ms 性能目标变成门禁；同时 stable-check 能检查 AI Bot 配置，但公网验收没有确认信号详情最终生成了 Web、AI 分析和提醒三个真实动作 URL。

## 修复

- 公开情报接口使用三个不同的一天窗口做冷缓存采样，以中位数验证 800ms SLO。
- 继续在同一轮检查未压缩响应不超过 256 KiB。
- 从最新信号取得稳定 `public_ref`，请求公开信号详情并验证精确信号深链。
- 验证 `ai_url` 与 `alert_url` 是否为 Telegram 深链；缺少用户名时明确给出配置警告。
- 单独检查公开信号详情总耗时不超过 800ms。
- `PUBLIC_SLO_MS` 可由环境变量调整，但默认值与工程优化计划保持一致。

## 部署影响

业务接口和页面不变。部署验收会增加四次只读公开 API 请求；性能超标会作为阻断，AI/提醒深链缺失会作为观察项，直到服务器填写 `AI_BOT_USERNAME`。
