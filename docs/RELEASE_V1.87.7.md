# v1.87.7 生产情报延迟收口

## 原因

v1.87.6 已完成 HTTPS、证书续期、动作深链和服务状态验收，但生产服务器仍有两个阻断：公开情报冷请求约 1.02 秒、信号上下文约 1.15 秒，超过工程计划确定的 800ms SLO。

生产数据只读剖析显示，2,000 条历史事实共约 6.3MB 文本，其中仅 296 条包含绝对规模标签；旧实现仍对全部文本执行 5,592 次正则扫描，单项占用约 0.62 秒。信号上下文还会顺序等待行情网络请求与本地情报计算。

## 修复

- 在绝对规模解析前增加精确标签存在性判断；没有任何受支持标签时跳过大文本正则扫描。
- 结构化字段仍具有最高优先级，文本解析、排名、生命周期和共振语义保持不变。
- 信号上下文并行加载行情快照与本地情报，让冷请求只承担较慢分支的耗时，而不是两者之和。
- 增加结构化/文本/无关长文本解析回归测试，以及行情与情报确实并发执行的同步屏障测试。

## 生产数据验证

在生产数据库的独立只读进程中验证：

- 2,000 条记录的新旧绝对规模解析结果完全一致，共识别 270 条有效指标。
- 情报核心计算中位数约 0.34 秒。
- 公开情报冷请求中位数约 0.49 秒，低于 800ms SLO。
- 并行信号上下文核心五次样本为 0.58–0.66 秒，中位数约 0.61 秒。

## 部署影响

不改变公开 API 契约、排名口径、数据库结构或外部动作 URL。升级后重启 Web 服务即可生效；完整效果以服务器 HTTPS 验收三次冷请求和信号上下文门禁为准。
