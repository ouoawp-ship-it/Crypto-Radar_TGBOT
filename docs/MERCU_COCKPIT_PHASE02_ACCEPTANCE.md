## 2. 数据基础

### 2.1 版本化资产分类

新增项目自维护资产分类，版本为 `2026.07.1`，首批包含：L1、L2、DeFi、Meme、AI、RWA、DePIN、GameFi、BTC 生态、支付、隐私、预言机、跨链、质押、交易平台币、稳定币和其他。

聚合规则：

- 每个资产可以有多个标签。
- 仅第一个主板块参与资金总额聚合。
- 附加板块仅用于筛选和解释，不重复计算资金。
- 未收录资产统一进入“其他”，不根据币名在运行时猜测分类。

### 2.2 主动买入、主动卖出与 CVD

资金流扫描现在同时保存：

- 现货主动买入成交额。
- 现货主动卖出成交额。
- 现货 CVD 净额。
- 合约主动买入成交额。
- 合约主动卖出成交额。
- 合约 CVD 净额。

这些数据来自 Binance 封闭 K 线窗口的 taker buy quote volume 和总成交额。它们表示主动买卖成交差，不代表交易所链上充提净流入。

新字段已通过 SQLite 自动迁移加入现有 `market_snapshots` 表，无需删除原数据库。旧快照仅有 CVD 净额时，买入/卖出额保持 `null`。

### 2.3 市值和数据源

批量行情快照在 Binance 公开币种目录可用时写入市值，无法匹配时返回 `null`。价格、成交额、OI、费率、资金流和市值分别返回数据源名称、更新时间和质量标记。

## 3. 资金中心 `/funds`

### 3.1 公开 API

- `GET /public-api/funds/sectors`
- `GET /public-api/funds/assets`

支持：

- 现货/合约切换。
- `15m / 30m / 1h / 4h / 1d` 窗口。
- 板块、币种、数据状态筛选。
- 净流入、涨跌幅、成交额、OI、资金费率和市值排序。
- 页码和最大 100 条的服务端分页。
- `schema_version`、`catalog_version`、`generated_at`、`coverage`、`data_status`、`warnings` 和 `methodology`。

### 3.2 页面

- 总净流入、主动买入额、主动卖出额、覆盖数和领先板块。
- 气泡面积表示绝对净额，颜色表示流入/流出。
- 移动端使用排序列表代替气泡，不依赖悬停。
- 资产表桌面端为高密度表格，移动端为证据卡片。
- 缺失排序值永远位于有效值之后。
- 资金流完全未覆盖时展示说明性空状态，不生成全零气泡。

## 4. 单币证据工作台 `/coin/[symbol]`

当前包含：

- 价格、24h 涨跌、成交额、市值、OI、费率摘要。
- 现货/合约 K 线切换。
- `1m / 5m / 15m / 1h / 4h / 1d` 周期。
- 最多 240 根、默认 96 根的服务端有界 K 线。
- 价格、OI、现货 CVD、合约 CVD 历史快照曲线。
- 自身历史排名、市场强度、绝对量级与生命周期。
- 多交易所资金费率。
- 已入库官方公告和同币信号时间线。
- 自选、只看该币信号、AI 分析、提醒和复制分享链接。

K 线数据不可用时不使用模拟数据。快照库未累积至少两个样本时，曲线区显示“样本不足”。

## 5. Web、Telegram 与 AI 上下文

- Web 的分享链接保留 `symbol` 和最新 `signal_ref`。
- AI 深链格式：`analyze_{symbol}_{signal_ref}`。
- 提醒深链格式：`alert_{symbol}_{signal_ref}`。
- Bot 同时兼容旧格式 `analyze_{symbol}` 和 `alert_{symbol}`。
- AI 币种档案只在 `signal_ref` 存在、引用可读且币种一致时加载该信号。
- 不匹配或不可读的引用不注入 AI 上下文。
- 从信号进入的提醒会在内部来源和备注中保留 `signal_ref`。

## 6. 自选闭环

- 自选页保留匿名、浏览器本地存储。
- 自选资产卡片增加现货 CVD、合约 CVD、1h OI 和资金费率。
- 空状态同时提供“去信号雷达”和“查看资金榜单”。
- 快照库没有该资产资金证据时，`flow` 为 `null`。
