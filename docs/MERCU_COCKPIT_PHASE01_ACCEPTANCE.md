## 2. Phase 0：数据与契约基础

### 2.1 市场快照存储

新增 SQLite 市场快照存储，支持：

- 批量写入与幂等更新。
- WAL 模式、必要索引和保留期裁剪。
- 按 `15m / 30m / 1h / 4h / 1d` 读取对比快照。
- 合并价格、持仓、资金费率、现货 CVD 和合约 CVD 等最新字段。
- 默认 5 分钟快照间隔与 7 天保留期，可通过环境变量配置。

### 2.2 市场指标语义

当前统一输出：

- 价格变化：优先使用指定时间窗口快照；历史不足时仅允许价格使用 24h 行情回退，并明确标记。
- OI 变化：指定窗口的持仓量变化；无历史数据时不伪造数值。
- 现货/合约主动资金：来自已完成资金流扫描的 CVD 估算，只在有效覆盖内参与榜单。
- 资金费率：展示当前周期的正费率和负费率极值。
- 横截面强度：在当次有效样本内按分位数归一化，不与绝对收益混用。

### 2.3 五类双向榜单

1. 价格涨幅 / 跌幅榜。
2. OI 增长 / 下降榜。
3. 合约主动流入 / 流出榜。
4. 现货主动流入 / 流出榜。
5. 资金费率正值 / 负值榜。

所有榜单都返回单位、时间窗口、覆盖资产数、数据状态和方法说明，空数据保持空态，不以 `0` 伪装有效计算。

### 2.4 公开 API

- `GET /public-api/market/overview`
- `GET /public-api/radar/boards`

API 具备 15 秒短缓存、输出数量上限、敏感字段脱敏和重型路由限流。公开健康状态也增加了市场历史快照的可用性信息。

## 3. Phase 1：三栏市场情报驾驶舱

### 3.1 页面结构

- 顶部：紧凑产品导航、LIVE 状态、浅色/深色主题和后台入口。
- 左栏：实时异动事件流，包含排名、生命周期、共振和证据详情入口。
- 中栏：价格、OI、合约资金、现货资金四组热钱榜单，以及现有机会看板。
- 右栏：全市场状态、资金倾向、数据覆盖和降级说明。

### 3.2 交互

- 币种、关键词、模块和生命周期筛选。
- `15m / 30m / 1h / 4h / 1d` 时间窗口切换。
- 30 秒后台刷新，新事件先进入待更新队列，避免用户阅读时列表突然跳动。
- 暂停/恢复实时刷新，支持手动刷新。
- 榜单币种可反向填入筛选；事件可打开原有证据与上下文抽屉。
- 筛选状态同步到 URL，便于复现和分享。
- 浅色/深色主题持久化到浏览器本地偏好。

### 3.3 响应式布局

- 1536px 及以上：`280px / 自适应主区 / 300px` 三栏。
- 1024px–1535px：事件流 + 主区双栏，市场概览改为三张横向卡片。
- 平板和手机：单栏信息流，保留筛选、榜单、详情抽屉和主操作。
- 360px、390px、768px 均已验证无水平溢出。

## 5. 当前预期降级状态

本地页面当前显示 `DEGRADED` 是预期行为，不是构建失败：

- 快照表刚创建，还没有累积足够的多时间窗口历史。
- 本地没有连续运行 OI 和资金流扫描，因此 OI、现货 CVD 和合约 CVD 榜单保持空态。
- 价格榜使用明确标注的 24h 行情回退，待快照积累后自动切换到真实窗口对比。

这种降级策略的原则是：不伪造精度，不把缺失值显示为有效零值，不因非关键榜单缺失阻断整个雷达页。
