# 产品边界：Telegram 信号雷达

## 定位

项目只负责把可解释、可追踪的加密市场信号推送到 Telegram。它不是网站、交易终端、用户平台、自动交易系统或 AI 聊天机器人。

## 信号链路

```text
交易所 REST / WebSocket / 可选 CoinGlass
                    ↓
标准化、缓存、限流与数据质量检查
                    ↓
价格 + OI + 成交量 + 资金费率 + 主动流 + 清算 + 公告
                    ↓
资金雷达 / 启动预警 / 费率警报 / 公告风险
                    ↓
去重、冷却、分级、话题路由、重试
                    ↓
Telegram
```

## 必须保持的原则

- 数据不足时明确降级，不能伪造或补零冒充真实数据。
- 单个指标不直接构成交易结论，推送必须展示触发依据。
- 默认 dry-run，真实发送需要双重确认与 readiness 门禁。
- BOT 计算层不依赖任何 Web 路由、浏览器或 Node.js 运行时。
- Web 产品若未来重建，应作为独立项目消费稳定的数据接口，不再嵌入本仓库。
