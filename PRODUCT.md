# Product

## Register

product

## Platform

web

## Users

主要用户是需要持续观察加密市场的交易员和 Telegram 信号订阅者。他们通常在桌面端同时关注大量异动、资金和资讯，在移动端快速复核推送信号；后台用户是维护数据源、服务、任务、密钥和推送策略的运营者。

## Product Purpose

Paoxx 将交易所衍生品、现货主动成交、链上实体资金流和价格结构整理为可解释的市场事实与风险信号，并把 Web 验证与 Telegram 推送串成同一证据闭环。成功意味着用户能在十秒内识别最值得关注的三至五个信号，并在一次点击内查看来源、新鲜度、排名、共振、生命周期和数据质量，而不被迫在噪声中寻找重点。

## Positioning

Paoxx 是以自有数据层和计算引擎驱动的四维资金行为雷达：先给出可核验的市场事实，再提供结构化解释与风险提示，但不执行交易，也不伪造缺失数据。CoinGlass 是主衍生品数据源，Coinalyze 用于交叉验证和历史补充，Arkham 提供链上实体语义，交易所公开接口承担最低延迟行情。

## Brand Personality

克制、精确、可信。界面语气冷静直接，优先展示事实、变化与风险，不营造夸张的收益暗示。

## Anti-references

- 不再逐像素复制 Mercu 或延续其品牌表达；只保留已经验证有价值的功能语义和工作流。
- 不做高饱和霓虹、装饰性玻璃拟态、交易赌场氛围或大面积无意义动效。
- 不用重复卡片、徽章和颜色制造虚假层级，不让后台像另一套产品。
- 不恢复个人空间、复杂账号、自动交易或通用研究平台。
- 不提供 AI 智选、AI 问答或模型生成的交易结论。

## Design Principles

1. 重点先于完整：首屏先回答“发生了什么、是否重要、证据是什么”。
2. 数据必须可追溯：来源、时间、新鲜度和降级状态与数值同等重要。
3. 前后台共享语言：同一套导航、排版、状态、表格、表单和反馈规范贯穿公开工作站与运维控制台。
4. 密度服务判断：高频信息保持紧凑，但通过分组、对齐和渐进展开控制认知负担。
5. 行动保持克制：分析、提醒、推送和配置入口清楚可见，同时始终保留风险边界。

## Explicit Scope Exclusions

- No theme switch. The visual theme selected for release is fixed across frontend and backend.
- No user-management module, personal profile, avatar menu, or account center.
- No favorites, watchlist, star controls, or saved-item workflow.
- No AI assistant, AI recommendation page, or AI-generated trading decision.

## Release Theme

The release theme is fixed dark: near-black canvas, graphite surfaces, high-contrast neutral text, and semantic color used only for direction, risk, warning, and freshness. Frontend and admin share the same tokens and component vocabulary.

## Data Pillars

1. Derivatives behavior: OI, funding, liquidations, long/short positioning, taker flow, order-book liquidity.
2. Spot behavior: active buy/sell flow, trade size, volume structure, and exchange-level divergence.
3. On-chain entities: exchange flows, projects, funds, market makers, whales, dormant wallets, and label confidence.
4. Price structure: K-line context, key levels, 2B/123/box structures, volatility, and liquidation proximity.

## Accessibility & Inclusion

以 WCAG 2.2 AA 为基线；正文与状态文本满足对比度要求，键盘焦点清晰，移动触控目标不小于 44px。涨跌和告警不能只依赖颜色表达，并尊重减少动态效果偏好。
